🛡️ Database Encryption

{{if .EncryptionEnabled}}
🔐
Database encryption is active

The database is encrypted at rest using AES-256-GCM with the server's encryption key. When Bloom shuts down, the database file is encrypted and the plaintext copy is removed. Periodic encrypted snapshots are taken every 5 minutes for crash resilience. Note: the server operator controls this key — for maximum privacy, self-host Bloom.

{{else}}
🔓
Database encryption is not enabled

Database-at-rest encryption is not enabled on this server. If you're self-hosting, set the ENCRYPTION_KEY environment variable (8+ characters) to enable AES-256-GCM encryption. The database file will be encrypted when Bloom is not running.

{{end}}
